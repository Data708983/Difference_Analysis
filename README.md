# Hello world
在RNA-seq数据分析中，**counts**、**TPM**、**FPKM/RPKM** 是三种常见的基因/转录本表达量度量方式。它们从不同角度对测序数据进行量化，适用于不同的分析场景。

---

### 1. Counts（原始计数）
- **定义**：比对到某个基因（或转录本）的**read（或fragment）数量**，是测序实验直接产生的整数计数。
- **特点**：
  - 原始、未归一化，受**测序深度**（sequencing depth）和**基因长度**影响。
  - 通常由featureCounts、HTSeq、STAR等工具从比对文件中计数得到。
- **在差异分析中的作用**：
  - **是大多数差异表达分析工具的推荐输入**（如DESeq2、edgeR、limma-voom）。
  - 这些工具基于计数数据的**负二项分布**模型，利用原始计数估计基因表达离散度，并通过内部归一化（如DESeq2的median of ratios）消除测序深度差异，从而进行统计检验。
  - 直接使用原始计数可保留数据的统计特性（整数、方差与均值相关），保证检验的有效性。

---

### 2. TPM（Transcripts Per Million，每百万转录本数）
- **定义**：首先对每个基因的计数**除以基因长度（kb）**得到RPK（reads per kilobase），再对某个样本中所有基因的RPK求和得到“每百万”缩放因子，最后用每个基因的RPK除以该缩放因子并乘以 \(10^6\)。
- **特点**：
  - **同时消除了测序深度和基因长度的影响**，使得不同样本间、不同基因间的表达量可以直接比较。
  - 公式：  
    \[
    \text{TPM}_i = \frac{\text{counts}_i / \text{length}_i}{\sum_j (\text{counts}_j / \text{length}_j)} \times 10^6
    \]
- **在差异分析中的作用**：
  - **不推荐直接作为差异分析输入**，因为TPM已经过样本内归一化，破坏了原始计数数据的离散度结构，导致统计模型（如DESeq2）假设失效。
  - 常用于**基因表达水平的可视化**（如热图、箱线图）、**跨样本或跨基因的比较**，以及作为某些机器学习模型的输入特征。

---

### 3. FPKM / RPKM（Fragments/Reads Per Kilobase per Million）
- **定义**：
  - **RPKM**（单端测序）：先计算每千碱基外显子每百万比对的read数。  
  - **FPKM**（双端测序）：与RPKM类似，但以fragment（一个fragment可能对应一对reads）为单位。
  - 公式：  
    \[
    \text{FPKM}_i = \frac{\text{counts}_i \times 10^9}{\text{length}_i \times \text{total\_counts}}
    \]
- **特点**：
  - 同样校正了基因长度和测序深度，但计算顺序与TPM略有不同（TPM先除长度再归一化，FPKM先归一化再除长度），导致TPM在不同样本间总和恒定，而FPKM总和随样本变化。
  - 目前TPM逐渐取代FPKM/RPKM成为更常用的长度归一化度量。
- **在差异分析中的作用**：
  - 与TPM类似，**不宜直接用于差异分析工具**，理由同上。

---

| 数据类型 | 是否适合直接差异分析 | 原因 |
|----------|----------------------|------|
| **Counts** | **是（推荐）** | 原始计数保留了统计特性，差异分析工具内置归一化和离散度估计。 |
| **TPM / FPKM** | **否** | 经过长度和深度归一化，破坏了数据的计数分布特性，直接使用会导致统计检验失真。 |

**实践建议**：
- 使用**featureCounts / HTSeq**等工具得到所有样本的**counts矩阵**，作为DESeq2/edgeR/limma的输入。
- 可将TPM用于表达量的**可视化**或**功能富集分析**（如GSEA，有时接受log2(TPM+1)），但差异分析仍以原始counts为准。
